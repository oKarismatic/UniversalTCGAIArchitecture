# CONTRIBUTING

Thank you for your interest in contributing to AI-PoweredTCGAnalysisFramework. This is a simple guide for non-coders and contributors.

What I accept
- Documentation improvements (clarity, typos, formatting)
- New prompt files or updated prompt versions (please include original filename and date)
- Screenshots and demo assets (place under docs/assets/ and reference them in README)

How to contribute
1. Fork the repository
2. Create a branch with a descriptive name (e.g., fix/typo-readme, add/prompt-dual-personality)
3. Make changes and open a Pull Request describing what you changed and why

Non-technical contributions
- Open Issues to track ideas or tasks and label them (idea, docs, housekeeping)
- Add a short entry to CHANGELOG.md describing your change

Style guidelines
- New Markdown files: use lowercase-with-hyphens for filenames
- Keep original .txt files intact — copy and convert to .md instead of deleting

If you’re unsure, open an Issue first and I’ll help triage.
